Ради всего святого, подключи линтер!

Линтер это инструмент, который предназначен для обеспечения соблюдения стиля кодирования.
В новых flutter проектах по умолчанию уже подключены flutter_lints, но стандартные правила из данного пакета не настолько строгие.
Для чего нужны строгие правила?
  • Новичок, по незнанию, может написать плохой код. Линтер подскажет как делать можно, а как лучше не стоит.
  • Любой разработчик может забыть что-то(например дописать ключевое слово const). Линтер об этом напомнит.
Рекомендуемые пакеты с более строгими правилами:
  • surf_lint_rules
  • very_good_analysis
  • lint

Не забудь исключить из правил линтера ненужные или сгенерированные файлы. Пример:

analyzer:
  exclude:
    # Build
    - "build/**"
    # Generated code
    - "**/*.g.dart"
    - "**/*.config.dart"
    - "**/*.freezed.dart"
